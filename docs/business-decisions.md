## 2026-06-27 — 사업자 구조, 결제 채널, 멀티 앱 전략, 언어 전략 일괄 확정

### 1. 사업자 / 법적 구조

- **사업자등록 1개로 향후 만들 모든 앱 운영 가능** (Omni Workspace, Think Tile, 웹소설 AI, 한의원 AI 등). 앱마다 사업자를 새로 낼 필요 없음
- 사업자등록은 **결제 채널과 무관하게 무조건 필요**
- **Windows에서 자체결제(Paddle 등)를 쓰는 순간 통신판매업 신고도 필요해짐.** Android/Apple 인앱결제만 썼다면 신고 불필요했을 부분 — Windows 직접결제 채널이 이 의무를 발생시킴
  - Android(Play Billing), Apple(StoreKit): 플랫폼이 Merchant of Record → 통신판매업 신고 불필요
  - Windows(Paddle/Polar 등 자체결제): 직접 판매 구조 → 통신판매업 신고 필요
- 출시 준비 순서: **사업자등록 → 통신판매업 신고 → Paddle/Polar 계약 → 개인정보처리방침 → 이용약관 → 환불정책**
- 해외 매출(외화) 관련: 일반적인 소액 구독 매출 규모에서는 큰 문제 없는 편이지만, 매출 규모가 커지면 외국환거래법상 신고 의무(예: 건당 일정 금액 초과 시) 검토 필요 — 세무사 확인 권장. **Apple/Google 인앱결제 채널은 이 부담이 거의 없음** (정산을 플랫폼이 대신 처리)

### 2. 결제 채널 (확정)

| 플랫폼 | 결제 방식 | 결제수단 | 비고 |
|---|---|---|---|
| Windows | **Paddle (또는 Polar)** | 카드 (+PayPal 가능하면) | Stripe 아님 — 아래 참고 |
| Android | Google Play Billing | Play 지원 수단 전체 (카드/통신사결제/Play잔액 등) | |
| Apple (Mac/iPad/iPhone) | App Store IAP | Apple 지원 수단 전체 (카드/Apple Pay 등) | |

- ⚠️ **Stripe는 한국 사업자로 직접 계정 개설 불가** (한국이 Stripe Merchant Country 아님, 2026-01 기준) → Paddle/Lemon Squeezy/Polar 같은 MoR(Merchant of Record) 서비스로 대체
- **Windows는 "인앱결제"라는 개념 자체가 없음** — `.exe` 직배포 시 결제 강제가 전혀 없어 Paddle 결제창을 앱 안에 webview로 그대로 띄우면 됨(수수료 없음). Microsoft Store를 통해 배포할 경우 MS 자체 결제(수수료 15%) 또는 자체결제(수수료 없음) 둘 다 선택 가능
- **Google Play Billing은 Android 전용** — Windows 데스크톱 앱에 "구글 인앱결제"를 넣는다는 개념 자체가 존재하지 않음 (Google Pay는 결제수단일 뿐 별개 개념)
- **Apple만 외부결제 우회를 강하게 막음** — Mac/iPad/iPhone 앱 안에서 Stripe/Paddle 같은 외부결제 링크 노출 시 심사 거부 가능성 큼. Windows(MS)는 이 정책이 느슨해서 문제 없음
- **검증 필요 (미검증)**: Paddle/Polar 결제 위젯이 Electron webview 안에서 실제로 뜨는지 — 결제 서비스의 안티프로드 휴리스틱이 webview를 일반 브라우저보다 더 깐깐하게 차단할 가능성 있음. Grok/Perplexity Google OAuth가 webview에서 막혔던 전례와 동일한 리스크 패턴. **출시 전 가장 먼저 작게 검증해볼 항목**
- Paddle 선택 시 트레이드오프: 세무 부담(VAT/소비세 계산, 영수증)을 Paddle이 대신 처리해주는 대신, 세금계산서 발행 불가 + 카카오페이/네이버페이 미지원 (카드결제만) — 한국 사용자 비중이 높을수록 이 약점이 아픔

### 3. Windows + Android 구독 통합

- **Windows(Paddle)와 Android(Play Billing)는 결제 채널이 서로 다른 곳에 등록됨** → 자동으로 동기화되지 않음
- **Omni Account(자체 로그인) + 서버(Supabase)에서 entitlement 통합 판단 필요.** 결제가 어느 채널(Paddle/Play Billing/StoreKit)에서 일어나든, 서버가 웹훅으로 결제 상태를 받아 같은 계정에 "Pro" 플래그를 찍어주는 구조 → 윈도우에서 결제한 사람이 안드로이드 앱 켜도 Pro로 인식됨
- 이 구조가 성립하려면 **Omni 자체 계정 시스템(이메일 또는 구글/애플 로그인)이 전제조건** — 아직 미설계, 다음 작업 후보

### 4. 멀티 앱 전략 (확정)

향후 Think Tile(노트 앱), 웹소설 AI, 한의원 AI 등 여러 앱을 만들 가능성을 염두에 두고 정리:

- **사업자 1개로 모든 앱 공유** (세금/매출 단위 — 법적으로 자동 묶임)
- **홈페이지(도메인) 1개로 모든 앱 공유** — 포트폴리오/랜딩 역할 (예: `huingilab.com/workspace`, `/thinktile`, `/novel` 형태)
- ~~계정 통합~~ → **계정/Auth/DB는 앱마다 완전히 독립.** 각 앱이 자체 Supabase 프로젝트, 자체 회원가입 시스템을 가짐
  - 이유: 라인업(멀티 AI 데스크톱 / 노트 / 웹소설 / 한의원)의 타겟 사용자층이 서로 거의 겹치지 않음 → 계정 통합의 핵심 가치("회원가입 없이 다른 앱도 써보기")가 실익이 적고, 오히려 불필요한 연결처럼 느껴질 수 있음
  - 사업자/홈페이지처럼 "법적·브랜드 차원에서 자동으로 묶이는 것"과 "계정처럼 직접 설계해야 하는 소프트웨어 아키텍처"는 별개 문제라는 점 주의 — 사업자등록을 한다고 계정 시스템이 자동으로 통합되는 게 아님
- **구독도 앱별로 완전 별도** (통합 멤버십 아님). 한 앱만 쓰는 사용자가 다른 앱 비용까지 묻어가는 구조를 피함
- → Omni Workspace의 `users`/`subscriptions` Supabase 스키마는 멀티프로덕트 고려 없이 **단독 설계로 단순하게 가면 됨** (한때 검토했던 `product_id` 필드 분리 구조는 불필요해짐)

### 5. 홈페이지 비용

- 도메인(.com): 연 약 14,000~29,000원
- 호스팅: Vercel/Netlify/Cloudflare Pages 무료 티어로 충분 (정적 포트폴리오 페이지 기준)
- **총 고정비: 연 2만원 안팎** — Apple Developer($99/년)나 코드사이닝 인증서 비용에 비하면 미미한 수준

### 6. 언어 / 시장 전략 (확정)

- **1차 출시 타겟: 한/영 동시 출시**
- 이 결정이 Windows 결제 채널 선택(Paddle, 영어권 친화적)을 사실상 확정시킴 — 한국 단독이었다면 카카오페이/네이버페이 미지원이 더 아쉬웠을 것
- i18n 구조: **지금 단계 코드부터 적용** (`react-i18next`, 텍스트 키-값 분리) — 기존 화면은 한 번에 다 바꾸지 않고, 새로 만드는 컴포넌트부터 키 방식으로 작성, 기존 화면은 나중에 일괄 리팩터링
- 홈페이지도 동일 패턴(`react-i18next`, 언어 prefix URL `/en/workspace`)으로 한/영 대응 필요 — 단, **지금 당장 할 일 아님, 출시 준비 단계에 진행**
- 한/영 동시 출시가 끌고 오는 부수 작업(코드 외): App Store/Play Store 등록정보 영문화, 이용약관/개인정보처리방침/환불정책 영문 버전, 영어 고객지원 체계 — 출시 직전 to-do로 별도 관리

## (기존 전체기획 문서 — 사업 관련 부분 발췌)

> 아래는 6/27 논의 이전, 프로젝트 초기 기획 문서(`전체기획`)에 있던 비용/정책 정보. 위 결정사항과 함께 참고.

### 플랫폼별 등록 비용 (전체기획 문서 기준, 변동 가능성 있어 출시 전 재확인 필요)

| 플랫폼 | 등록 비용 | 비고 |
|---|---|---|
| Microsoft Store | 무료 (개인 2025-09부터, 회사 2026-05부터 등록비 폐지) | 직접배포(.exe)는 등록비 자체가 없음 |
| 코드사이닝 인증서 (.exe 직배포 시) | OV 기준 연 $200~300, 또는 클라우드 방식 월 $9.99부터 | **유효기간이 2026년 초부터 약 15개월로 단축됨** — 매년 갱신 일정 캘린더 등록 필요 |
| Google Play | $25 (1회성) | |
| Apple Developer Program | 연 $99 | |

### 가격/티어 정책 (확정된 부분 — 코드에 반영된 현재 값)

- 상단탭(동시에 열린 탭, 일반그룹+워크스테이션 합산): 무료 2 / 프로 4
- 워크스테이션 총 보유 개수: 8개 고정
- 슬롯/분할: 무료 슬롯 2개·분할 2까지 / 프로 슬롯 8개·분할 4까지 (가로4분할 ↔ 상2하2 전환)
- 유료 전용 기능: 워크스테이션 생성/저장, 메모 기능

### 보류된 초기 수익화 아이디어 (참고용, 위 확정값으로 대체됨)

- 프로젝트 초기에는 "무료 1~2개 워크스페이스 / 프로 최대 10개"로 잠정 구상했었으나, 이후 "워크스테이션 8개 고정 + 상단탭 무료2/프로4"로 구체화되며 대체됨
